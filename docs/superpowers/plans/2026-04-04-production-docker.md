# Production-Ready Docker Deployment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the PDF-to-CSV app self-hostable on a VPS behind Caddy with a lean Docker image, bundled PDF.js worker, and a one-command deploy script.

**Architecture:** Next.js standalone output build served from a Docker container on port 3210 internally, mapped to host port 3210. Caddy sits in front with HTTPS and IP allowlist. PDF.js worker served as a local static asset instead of loading from CDN.

**Tech Stack:** Next.js 14, pdfjs-dist 3.x, Docker (multi-stage build), docker compose, Caddy

## Task 3: Rewrite Dockerfile

**Files:**
- Modify: `Dockerfile`

- [ ] **Step 1: Replace Dockerfile contents**

The current Dockerfile has markdown garbage after `CMD ["npm", "start"]` from a bad paste. Replace the entire file contents with:

```dockerfile
FROM node:18-alpine AS builder
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:18-alpine
WORKDIR /app
RUN apk add --no-cache dumb-init
RUN addgroup -g 1001 -S nodejs && adduser -S nextjs -u 1001
COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static
COPY --from=builder --chown=nextjs:nodejs /app/public ./public
USER nextjs
EXPOSE 3210
ENTRYPOINT ["dumb-init", "--"]
CMD ["node", "server.js"]
```

Key changes from the old file:
- Removes all markdown content after `CMD`
- Production stage copies from `.next/standalone` (not `.next` directly)
- Also copies `.next/static` and `public` alongside the standalone output
- Uses `node server.js` (standalone entrypoint) instead of `npm start`

- [ ] **Step 2: Commit**

```bash

git commit -m "fix: rewrite Dockerfile to use standalone output, remove corrupted content"
```

## Task 5: Fix docker-compose.yml

**Files:**
- Modify: `docker-compose.yml`

- [ ] **Step 1: Replace docker-compose.yml contents**

Replace the entire file with:

```yaml
version: '3.8'

services:
  pdf-to-csv:
    build:
      context: .
      dockerfile: Dockerfile
    container_name: pdf-to-csv
    ports:
      - "3210:3210"
    env_file:
      - .env
    environment:
      - NODE_ENV=production
      - PORT=3210
      - HOSTNAME=0.0.0.0
    restart: unless-stopped
    networks:
      - pdf-network
    healthcheck:
      test: ["CMD", "wget", "-qO-", "http://localhost:3210"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s

networks:
  pdf-network:
    driver: bridge
```

Key changes from the old file:
- Port changed from `3210:3210` to `3210:3210` (container runs on 3210, host accesses via 3210)
- Added `env_file: .env` for local config
- Added `HOSTNAME=0.0.0.0` — required for Next.js standalone to bind on all interfaces inside the container (otherwise it only binds on `127.0.0.1` and is unreachable from outside the container)
- Added `PORT=3210` so Next.js standalone picks up the correct port
- Fixed healthcheck URL from `http://localhost:3210` to `http://localhost:3210` (inside the container, the app is on 3210)
- Switched healthcheck from `curl` to `wget` — Alpine doesn't include curl by default but has wget

- [ ] **Step 2: Commit**

```bash
git add docker-compose.yml
git commit -m "fix: correct port mapping, healthcheck URL, and add env_file to docker-compose"
```

These steps are run **on the VPS**, not locally.

```

```

No edits needed — the defaults are correct for production.

- [ ] **Step 3: Make deploy.sh executable**

```bash
chmod +x deploy.sh
```

- [ ] **Step 4: Build and start the container**

```bash
docker compose up --build -d
```

- [ ] **Step 5: Add the Caddy site block**

Open your Caddyfile (typically `/etc/caddy/Caddyfile` or wherever your Caddy config lives) and add:

```caddyfile
pdf.yourdomain.com {
    @allowed remote_ip YOUR.IP.HERE
    handle @allowed {
        reverse_proxy localhost:3210
    }
    handle {
        respond "Access denied" 403
    }
}
```

Replace `pdf.yourdomain.com` with your actual subdomain and `YOUR.IP.HERE` with your public IP.

- [ ] **Step 6: Reload Caddy**

```bash
caddy reload
# or if using systemd:
systemctl reload caddy
```

- [ ] **Step 7: Verify HTTPS access**

Open `https://pdf.yourdomain.com` in the browser. Confirm:
- Page loads with the PDF to CSV UI
- PDF upload and conversion works
- Any IP not on the allowlist gets a `403 Access denied` response
